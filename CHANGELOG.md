# Changelog

## [Unreleased]

### Исправлено

- `FileBase64Rule`/`Base64Rule` больше не бросают исключений на не-строковом вводе (массив/число в payload) — значение проваливает валидацию с base64-сообщением вместо `TypeError` (HTTP 500).
- Извлечение payload из data-URI переписано на одну регулярку: легальные заголовки с дополнительными параметрами (`data:image/png;charset=utf-8;base64,...`) и заголовки без разделителя payload больше не падают с `Undefined array key` — они валидируются.
- Сообщения, ссылающиеся на `:cur_width`/`:cur_height`, когда payload не декодируется в изображение, придерживаются: сырые плейсхолдеры не утекают пользователю, провал сообщают остальные правила (image/mimetypes).
- Не-base64 payload отсекается сразу с base64-сообщением — раньше записывался пустым временным файлом и вводил в заблуждение провалом image-правила.
- Временный файл удаляется в `finally`, а сбой `tempnam()` бросает `RuntimeException`, вместо продолжения с битым путём.

### Изменено

- **URL-пропуск сужен до `http`/`https`.** Ранее любой `filter_var`-валидный URL любой схемы (`ftp://`, `phar://`, ...) молча пропускал файловую валидацию целиком.
- `:hint` подставляется пустой строкой при отсутствии значения соседнего поля (ранее в замены уходил `null`).
- `getHint()` резолвит позиции wildcard'ов только из числовых сегментов пути атрибута (цифры внутри имён полей игнорируются).

### Добавлено

- `LaravelValidationBase64ServiceProvider` с переводами `base64::validation.base64` (`en`, `ru`), публикация через `--tag=base64-lang`.
- Тестовый каркас (orchestra/testbench) на оба правила, GitHub Actions CI (PHP 8.4/8.5 × Laravel 12/13 + pint), `laravel/pint`.
- `.gitattributes` с `export-ignore` — dist-архивы уезжают без тестов/CI/dev-файлов.
- `declare(strict_types=1)` во всём пакете.
